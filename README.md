# Mind • Matter • Meaning

Mind • Matter • Meaning is a static academic website built with Jekyll, Markdown, Liquid templates, HTML5, CSS3, and vanilla JavaScript. It is designed for GitHub Pages and intentionally avoids a backend, database, React, Vue, Angular, Bootstrap, paid services, trackers, cookies, and third-party CMS software.

## Architecture

- `_config.yml` defines site metadata, Jekyll collections, default layouts, plugins, and permalink rules.
- `_layouts/` contains the default, page, post, category, and tag templates.
- `_includes/` contains reusable fragments for the head, navigation, footer, reading time, related articles, and post cards.
- `_posts/` stores essays as Markdown files named `YYYY-MM-DD-title.md`.
- `_categories/` and `_tags/` provide optional landing pages for durable taxonomy URLs.
- `_data/books.yml` stores book data used by `books.md`.
- `assets/css/main.css` contains the complete visual system.
- `assets/js/main.js` provides dark mode, mobile navigation, reading progress, back-to-top behavior, and client-side search.

## How Jekyll Works

Jekyll reads Markdown files with YAML front matter, applies Liquid templates from `_layouts/` and `_includes/`, and emits a static `_site/` directory. GitHub Pages can build this automatically on every push to `main`.

## How GitHub Pages Works

The workflow in `.github/workflows/pages.yml` uses official GitHub Pages actions:

1. `actions/checkout`
2. `actions/configure-pages`
3. `actions/jekyll-build-pages`
4. `actions/upload-pages-artifact`
5. `actions/deploy-pages`

In the repository settings, set Pages source to GitHub Actions.

## Markdown Articles

Create a new essay in `_posts/`:

```markdown
---
title: The Nature of Consciousness
subtitle: Mind, matter, and explanation.
date: 2026-07-01
last_updated: 2026-07-01
categories: [philosophy]
tags: [consciousness, mind, reality]
---
## Introduction

Article text begins here.
```

Each Markdown file automatically becomes a complete webpage with title, subtitle, author, publication date, last updated date, reading time, category, tags, table of contents, related articles, previous/next links, and license metadata.

## Categories and Tags

Posts declare categories and tags in front matter. Archive, category, tag, related-article, and search views are generated from those values using Liquid.

## Search

`search.json` is generated by Jekyll from the post archive. `assets/js/main.js` fetches it and performs client-side search in the browser. No external API is used.

## Dark Mode

Dark mode is implemented with CSS custom properties and a small localStorage toggle. Change colors in `assets/css/main.css`.

## MathJax

MathJax is loaded in `_includes/head.html`. Inline math uses `$...$` or `\\(...\\)`. Display math uses `$$...$$` or `\\[...\\]`.

## Fonts and Colors

Google Fonts are loaded from `_includes/head.html`: Libre Baskerville for headings, Inter for body text, and JetBrains Mono for code. Colors are controlled by CSS variables in `assets/css/main.css`.

## Images

Place optimized images in `assets/images/`. Use Markdown image syntax with useful alt text:

```markdown
![Phase diagram for a conceptual model]({{ '/assets/images/phase-diagram.png' | relative_url }})
```

## Books

Add or edit books in `_data/books.yml`. The `books.md` page renders the data automatically.

## New Pages

Create a Markdown file at the project root with front matter:

```markdown
---
title: New Page
permalink: /new-page/
---
```

Add navigation items in `_data/navigation.yml`.

## Bibliography and References

Each article should include `## References` and `## Further Reading`. For larger bibliographies, add data files under `_data/` and render them with Liquid.

## Publishing Updates

1. Edit Markdown, data, CSS, or template files.
2. Commit the change.
3. Push to `main`.
4. GitHub Actions builds and deploys the site automatically.

## Maintenance Principles

- Keep all long-form content in Markdown.
- Keep reusable structure in layouts and includes.
- Keep book lists and future structured data in `_data/`.
- Avoid plugins unsupported by GitHub Pages.
- Prefer accessible semantic HTML and durable URLs.
